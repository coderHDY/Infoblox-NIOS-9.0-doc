You can do the following after the initial setup, including uploading the initial ruleset:

- Review the list of DNS Infrastructure Protection members, as described in *[Listing Members](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280766329)*.
- Look at rulesets that are currently installed on your system, as described in *[Viewing DNS Infrastructure Protection Rules](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280766170)*.
- Upload new rulesets to the system when you have selected to manually apply rule updates, as described in *[Manually Uploading Rulesets](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280408834)*.
- Publish rule updates that you have uploaded to the system, as described in *[Publishing Rule Updates](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280408857)*.
- Modify the **Comment** and **Do** **Not** **Delete** parameters for an existing ruleset, as described in *[Modifying Rulesets](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280766372)*.
- View differences between an old and a new rulesets and optionally merge rule parameter changes from the old ruleset into the new one, as described in *[Comparing and Merging Rulesets](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280668919)*.
- Configure proxy settings for the Grid if you need to use a proxy server for ruleset updates, as described in *[Configuring Proxy Servers](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280765075)*.
- Delegate ruleset updates to a Grid member and select an alternate interface for downloading the updates, as described in Configuring Members and Interfaces for Automatic Updates, see *[Configuring Proxy Servers](https://infoblox-docs.atlassian.net/wiki/spaces/nios90/pages/280765075)*.
