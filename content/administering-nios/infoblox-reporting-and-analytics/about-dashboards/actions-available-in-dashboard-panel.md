In the **Dashboard** panel, you can do the following using the **Edit** drop-down list:

- Edit panels as described in the Editing Dashboards section below*.*
- Edit source, title, and description.
- Edit permissions as described in the Editing Permissions section below.
- Clone a dashboard as described in the Cloning Dashboards section below.
- Reset dashboards, as described in the Resetting Dashboards section below*.*

> **warning**
>
> ### Note
> 
> - You cannot set any dashboard as the home dashboard. The **Set as Home Dashboard** option available in the **Edit** drop-down list does not add a dashboard to the **Home Dashboard** tab.
> - The Splunk documentation states that a version attribute setting of 1.1 is necessary for all XML dashboards to be compatible with jQuery 3.5 or higher. The default Infoblox Reporting dashboards are updated to incorporate version 1.1. However, after upgrading to NIOS 9.0.4, the following warning may be displayed for your custom Reporting dashboards: This dashboard version is missing. Update the dashboard version in source.  
>   The **jQuery Upgrade Dashboards** in the **Reporting **> **Administration** tab provides comprehensive instructions to identify affected dashboards and ensures their compatibility with jQuery 3.5 or higher.
