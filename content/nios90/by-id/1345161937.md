---
title: "Adding IP Addresses to Existing Host Records"
source: "/space/nios90/1345161937"
pageId: "1345161937"
---
You can add unused and unmanaged addresses, including all their information, to existing host records. When you add an unmanaged address to a host record, the appliance adds the discovered data to the host record. You can select the desired host to which you want to add the unmanaged address.

You must specify a MAC address for each new IP address that you assign to the host record irrespective of whether the DHCP option is enabled or not.  
To add an unmanaged IP address to an existing host record:

1. From the IP Map or List panel, select an IP address, and then click **Add** -&gt; **Add** **to** **Existing** **Host** from the Toolbar.
2. In the *Select* *Host* dialog box, do the following:
   
   - In the table, select the host to which you want to add the selected IP address. You can also use the filters or the **Go** **To** field to narrow down the host list.    - Click the Select icon.  
     Grid Manager displays the *Host* *Record* editor. For more information, see [*About Host Records*](/nios90/infoblox-nios-9-0-x/administering-nios/ip-address-management/managing-ip-addresses/about-host-records).
3. In the *Host* *Record* editor, update the host properties.
4. Save the configuration and click **Restart** if it appears at the top of the screen. To close the editor without saving the changes, click the **Close** icon.
